## 【1】collections.ChainMap 字典链表

"""
collections.ChainMap 是 Python 标准库中 collections 模块提供的一个数据类型，它用于将多个字典或映射链接在一起形成一个逻辑上的整体。

ChainMap 提供了一种方便的方式来同时访问和操作这些链接的字典，它会按照链接的顺序依次查找键。可以将 ChainMap 理解为一个字典的链表，每个字典都是链表中的一个节点。当在 ChainMap 中查找键时，它会从链表的头部开始依次查找，直到找到第一个匹配的键为止。如果键在多个字典中都存在，则返回第一个匹配的键对应的值。
"""
import collections

# 创建两个字典
dict1 = {'a': 1, 'b': 2}
dict2 = {'b': 3, 'c': 4}

# 创建 ChainMap
chain_map = collections.ChainMap(dict1, dict2)
# 访问键 'a' 的值
print(chain_map['a'])  # 输出: 1
# 访问键 'b' 的值
print(chain_map['b'])  # 输出: 2（在 dict1 中找到了匹配的键 'b'）
# 访问键 'c' 的值
print(chain_map['c'])  # 输出: 4（在 dict2 中找到了匹配的键 'c'）
# 修改键 'b' 的值
chain_map['b'] = 5  # 修改后，字典dict1也会被该改变
# 查看修改后的值
print(chain_map['b'])  # 输出: 5
# 遍历 ChainMap 中的所有键值对
for key, value in chain_map.items():
    print(key, value)

print("dict1:",dict1)
print("dict2:",dict2)

"""
【2】collections.Counter
class collections.Counter([iterable-or-mapping ])
Counter 是dict 的子类，用于计数hashable 对象。它是一个多项集，元素存储为字典的键而它们
的计数存储为字典的值。计数可以是任何整数，包括零或负的计数值。
"""
from collections import Counter

# 创建 Counter 对象
c = Counter(['a', 'b', 'a', 'c', 'b', 'a'])
# 计数结果
print(c)  # Counter({'a': 3, 'b': 2, 'c': 1})
# 访问元素计数
print(c['a'])  # 3
# 计数更新
c.update(['a', 'b', 'd'])
print(c)  # Counter({'a': 4, 'b': 3, 'c': 1, 'd': 1})
# 合并计数器
c2 = Counter(['a', 'b', 'b', 'c'])
c += c2
print(c)  # Counter({'a': 5, 'b': 5, 'c': 2, 'd': 1})
# 常见元素
print(c.most_common(3))  # [('a', 5), ('b', 5)]


c = Counter(a=4, b=2, c=0, d=-2)
res = sorted(c.elements()) # ['a', 'a', 'a', 'a', 'b', 'b']
print(res)
"""
【3】collections.deque 
collections.deque是 Python 标准库中的一个双向队列（double-ended queue）类，它提供了高效的插入和删除操作，可以在队列的两端进行操作。

collections.deque 类的一些重要特性和用法：

1-双向队列：deque 对象是一个双向队列，支持从队列的两端进行插入和删除操作。
--在队列的左侧使用 appendleft() 方法插入元素，在右侧使用 append() 方法插入元素。
--可以在左侧使用 popleft() 方法移除元素，在右侧使用 pop() 方法移除元素。

2-限制队列长度：可以通过指定 maxlen 参数来创建一个有限长度的队列。
一旦队列达到最大长度，在插入元素时会自动移除队列的最早元素。如果不指定 maxlen 参数，则队列可以无限增长。

3-高效操作：deque 对象的插入和删除操作在两端都是高效的，时间复杂度为 O(1)。
这使得它在需要频繁插入和删除元素的场景中比列表更加高效。

3-可迭代性：deque 对象可以像列表一样进行迭代操作
可以使用 for 循环遍历队列中的元素，也可以使用切片等操作
"""
from collections import deque

# 创建空的 deque 对象
d = deque()

# 在右侧插入元素
d.append('a')
d.append('b')
d.append('c')

# 在左侧插入元素
d.appendleft('z')

# 遍历 deque 对象
for item in d:
    print(item)  # 输出：'z', 'a', 'b', 'c'

# 移除右侧元素
d.pop()
print(d)  # deque(['z', 'a', 'b'])

# 移除左侧元素
d.popleft()
print(d)  # deque(['a', 'b'])

# 创建有限长度的 deque 对象
d = deque(maxlen=3)
d.append(1)
d.append(2)
d.append(3)
d.append(4)  # 自动移除最早的元素 1
print(d)  # deque([2, 3, 4])

"""
【4】collections.defaultdict
collections.defaultdict 是 Python 标准库中的一个字典类，它是 dict 的子类。
与普通的字典不同，defaultdict 允许指定一个默认值（default value），以便在访问不存在的键时返回该默认值，而不会引发 KeyError 异常。

collections.defaultdict 类的一些重要特性和用法：
--1、默认值工厂函数：defaultdict 接受一个工厂函数作为参数，在访问不存在的键时会调用该工厂函数以生成默认值。
工厂函数可以是任何可调用对象，如内置函数、自定义函数或类的构造函数。

--2、自动创建键：如果访问一个不存在的键，defaultdict 会使用默认值工厂函数生成一个默认值，并将其与该键关联起来。
这样，即使字典中原本不存在该键，也可以直接访问并获取默认值。

--3、方便使用：defaultdict 的行为与普通字典完全一致，因此你可以像使用普通字典一样使用它。
它支持常见的字典操作，如访问键、设置键值对、删除键等。
"""

from collections import defaultdict

# 创建默认值为 0 的 defaultdict 对象
d = defaultdict(int)

# 访问不存在的键，默认值为 0
print(d['a'])  # 输出: 0

# 自动创建键，并更新默认值
d['a'] += 1
d['b'] += 2
print(d)  # defaultdict(<class 'int'>, {'a': 1, 'b': 2})

# 创建默认值为空列表的 defaultdict 对象
d = defaultdict(list)

# 访问不存在的键，默认值为空列表
print(d['a'])  # 输出: []

# 自动创建键，并添加元素到列表中
d['a'].append(1)
d['b'].append(2)
print(d)  # defaultdict(<class 'list'>, {'a': [1], 'b': [2]})

"""
【5】collections.namedtuple()
collections.namedtuple() 是 Python 标准库中的一个函数，用于创建具有命名字段的元组（named tuple）。
它提供了一种方便的方式来定义具有可读性和自描述性的数据结构。
collections.namedtuple() 的一些重要特性和用法：
--1、定义命名元组：namedtuple() 
接受两个参数，第一个参数是命名元组的名称，第二个参数是元组字段的名称，可以是字符串或字符串列表。
它返回一个新的元组子类，该子类具有指定的名称和字段。

--2、可读性和自描述性：
通过为元组字段命名，可以明确指定每个字段的含义和用途，从而提高代码的可读性和自描述性。
可以通过字段名称而不是索引来访问元组的值。

--3、字段访问：
创建的命名元组类具有与指定字段名称相对应的属性和方法。可以使用点符号访问字段的值，也可以使用索引访问。

--4、不可变性：命名元组是不可变的，即创建后不能修改元组的值。这与普通元组的行为相同。
"""
from collections import namedtuple

# 定义命名元组类 Point，包含字段 x 和 y
Point = namedtuple('Point', ['x', 'y'])

# 创建 Point 类的实例
p1 = Point(1, 2)

# 访问字段的值
print(p1.x)  # 输出: 1
print(p1.y)  # 输出: 2

# 使用索引访问字段的值
print(p1[0])  # 输出: 1
print(p1[1])  # 输出: 2

# 创建另一个 Point 类的实例
p2 = Point(3, 4)

# 字段访问和索引访问是等价的
print(p2.y)  # 输出: 4
print(p2[0])  # 输出: 3

# 命名元组是不可变的
# p1.x = 5  # 会引发 AttributeError

# 迭代命名元组的字段
for field in p1:
    print(field)  # 输出: 1 2


"""
【6】collections.OrderedDict
collections.OrderedDict 是 Python 标准库中的一个字典类，它是一个有序字典，可以按照元素添加的顺序进行迭代。
与普通的字典不同，OrderedDict 会记住元素的添加顺序，并在迭代时按照该顺序返回键值对。

collections.OrderedDict 的一些重要特性和用法：
--1、保持元素顺序：OrderedDict 会记录元素的添加顺序，因此在迭代字典时，键值对的顺序将与添加顺序保持一致。
--2、可用作 LRU 缓存：由于 OrderedDict 保留了元素的添加顺序，它可以用于实现 LRU（最近最少使用）缓存。
通过设置最大容量，当字典达到容量上限时，最早添加的元素将被自动删除。
--3、与普通字典兼容：OrderedDict 的行为与普通字典完全一致，因此你可以像使用普通字典一样使用它。它支持常见的字典操作，如访问键、设置键值对、删除键等。
"""

from collections import OrderedDict

# 创建有序字典
d = OrderedDict()

# 添加键值对
d['a'] = 1
d['b'] = 2
d['c'] = 3

# 迭代有序字典
for key, value in d.items():
    print(key, value)
# 输出:
# a 1
# b 2
# c 3

# 访问键值对
print(d['b'])  # 输出: 2

# 删除键值对
del d['a']
print(d)  # OrderedDict([('b', 2), ('c', 3)])

# 作为 LRU 缓存使用
d = OrderedDict()
d['a'] = 1
d['b'] = 2
d['c'] = 3
d.move_to_end('a')  # 将 'a' 移动到字典的末尾
d['d'] = 4  # 添加新元素，可能导致最早添加的元素被删除
print(d)  # OrderedDict([('b', 2), ('c', 3), ('d', 4)])

# 创建有序字典时保持元素顺序
d = OrderedDict([('a', 1), ('b', 2), ('c', 3)])
print(d)  # OrderedDict([('a', 1), ('b', 2), ('c', 3)])

"""
collections.UserDict 是一个方便的基类，用于创建自定义字典类。
它是 Python 标准库中的一个容器类，提供了一个可继承的字典实现，使创建自定义字典变得更加简单和灵活。
UserDict 的主要目的是作为自定义字典类的基类，通过继承它，可以轻松地扩展和定制字典类的行为。
相比直接继承内置的 dict 类，使用 UserDict 可以避免一些潜在的问题，并提供了一些方便的方法和属性。

collections.UserDict 的一些重要特性和用法：
--1、可继承性：UserDict 提供了一个可继承的字典实现，用于创建自定义的字典类。
通过继承 UserDict，可以轻松地自定义字典类的行为，添加额外的方法和属性，以满足特定的需求。
--2、更安全的修改：与直接继承 dict 不同，UserDict 的内部实际上使用了一个普通的字典对象来存储数据，而不是继承 dict。
这意味着在修改字典时，UserDict 中的方法会直接操作存储数据的字典对象，从而避免了一些潜在的问题，如无限递归。
--3、方便的扩展：UserDict 提供了一些方便的方法和属性，
如 data 属性用于访问底层的字典对象，keys()、values()、items() 方法用于返回字典的键、值和键值对。
这些方法和属性使得自定义字典类的扩展更加简单和一致。
"""
from collections import UserDict

class MyDict(UserDict):
    def __setitem__(self, key, value):
        # 在设置键值对时，将值都转换为大写
        super().__setitem__(key, value.upper())

    def get_keys(self):
        # 返回字典的键列表，按照字母顺序排序
        return sorted(self.keys())

# 创建自定义字典对象
my_dict = MyDict()

# 添加键值对
my_dict['a'] = 'apple'
my_dict['b'] = 'banana'
my_dict['c'] = 'cherry'

# 打印字典的键列表
print(my_dict.get_keys())  # 输出: ['A', 'B', 'C']

# 打印字典的键值对
for key, value in my_dict.items():
    print(key, value)
# 输出:
# a APPLE
# b BANANA
# c CHERRY

"""
如何理解无限递归？
"""
class BadDict(dict):
    def __setitem__(self, key, value):
        self[key] = value.upper()  # 错误的做法，会导致无限递归
    """
    在上面的代码中，我们定义了一个名为 BadDict 的类，它直接继承了内置的 dict 类，并重写了 __setitem__() 方法。在这个方法中，
    我们试图将键对应的值转换为大写，并将其保存到字典中。然而，这个实现是错误的。当我们在 __setitem__() 方法内部通过 self[key] 赋值时，
    实际上会再次调用 __setitem__() 方法，这样就形成了无限递归。每次调用 __setitem__() 方法都会导致新的递归调用，最终导致栈溢出错误。
    这是因为直接继承 dict 类时，我们在方法内部使用了相同的方法名 __setitem__()，从而覆盖了父类的方法。
    当我们尝试通过 self[key] 进行赋值时，会再次调用自身的 __setitem__() 方法，而不是父类的方法
    """


"""
collections.UserList 
是 Python 标准库中的一个容器类，它提供了一个可继承的列表实现，用于创建自定义的列表类。
UserList 允许开发人员通过继承来定制列表类的行为，并提供了一些方便的方法和属性。

collections.UserList 的一些重要特性和用法：
--1-可继承性：UserList 提供了一个可继承的列表实现，用于创建自定义的列表类。
通过继承 UserList，可以轻松地自定义列表类的行为，添加额外的方法和属性，以满足特定的需求。
--2-更安全的修改：与直接继承内置的 list 类不同，UserList 的内部实际上使用了一个普通的列表对象来存储数据，而不是继承 list。
这意味着在修改列表时，UserList 中的方法会直接操作存储数据的列表对象，从而避免了一些潜在的问题，如无限递归。
--3-方便的扩展：UserList 提供了一些方便的方法和属性，
如 data 属性用于访问底层的列表对象，append()、extend()、insert() 等方法用于修改列表的内容。这些方法和属性使得自定义列表类的扩展更加简单和一致。
"""
from collections import UserList

class MyList(UserList):
    def append(self, item):
        # 在添加元素时，将元素转换为大写
        super().append(item.upper())

    def get_unique(self):
        # 返回列表中的唯一元素
        return list(set(self.data))

# 创建自定义列表对象
my_list = MyList()

# 添加元素
my_list.append('apple')
my_list.append('banana')
my_list.append('cherry')
my_list.extend(['apple', 'date', 'banana'])

# 打印列表
print(my_list)  # 输出: ['APPLE', 'BANANA', 'CHERRY', 'APPLE', 'DATE', 'BANANA']

# 打印列表中的唯一元素
print(my_list.get_unique())  # 输出: ['APPLE', 'CHERRY', 'BANANA', 'DATE']

"""
UserString 是一个可继承的字符串类，它允许开发人员通过继承来创建自定义的字符串类，并添加额外的行为和功能。
通过继承 UserString，我们可以以更灵活的方式处理字符串数据，而无需直接操作字符串对象。

UserString 的一些重要特性和用法：

--1、可继承性：UserString 提供了一个可继承的字符串实现，使开发人员能够创建自定义的字符串类。
通过继承 UserString，可以轻松地自定义字符串类的行为，添加额外的方法和属性，以满足特定的需求。

--2、安全的修改：与直接继承内置的字符串类不同，UserString 内部使用了一个普通的字符串对象来存储数据，而不是继承字符串类。
这样，在修改字符串时，UserString 中的方法会直接操作存储数据的字符串对象，从而避免了潜在的问题。

--3、方便的扩展：UserString 提供了一些方便的方法和属性，如 data 属性用于访问底层的字符串对象，

from collections import UserString

class MyString(UserString):
    def reverse(self):
        # 返回字符串的反转版本
        return self.data[::-1]

    def count_vowels(self):
        # 返回字符串中元音字母的数量
        vowels = 'aeiou'
        count = 0
        for char in self.data.lower():
            if char in vowels:
                count += 1
        return count

# 创建自定义字符串对象
my_string = MyString("Hello, World!")

print(my_string.count_vowels())  # 输出: 3
