# Python 正则表达式

# 一、正则表达式概述

  1. 概念 正则表达式是一种用于操作字符串的逻辑公式，它由事先定义好的特定字符及这些字符的组合构成一个“规则字符串”，用于表达对字符串的过滤逻辑。

  2. 用途

  • 匹配和查找文本：能够快速定位包含特定模式的文本，如查找特定单词的句子、识别邮箱和电话等。
  • 数据清洗和处理：帮助去除特定字符或标签，提取有效信息。
  • 表单验证：用于校验用户输入的表单数据，如验证邮箱地址的合法性。
  • 字符串替换：将符合特定模式的字符串替换为其他内容。
  • 提取信息：从文本中提取所需的关键信息，例如网页中的链接、日志中的重要内容。

# 二、Python 中的正则表达式

Python 内置了 re 模块，其中封装了一系列用于处理正则表达式的函数。以下是一个简单示例：

import re

raw_str = 'abc12def'
result = re.search('\d+', raw_str)
print(result.group())  # 12

# 三、re 模块常用函数

  1. re.match 该函数尝试从字符串的起始位置匹配一个模式，如果起始位置不匹配，match 函数将返回 None。其函数签名为 re.match(pattern, string, flags=0) 。

  • 参数说明：

    • pattern：要匹配的正则表达式。
    • string：待匹配的字符串。
    • flags：标志位，用于控制匹配方式，如是否区分大小写、多行匹配等。

  • 常见标志：

    • re.I：使匹配对大小写不敏感。
    • re.L：进行本地化识别匹配。
    • re.M：支持多行匹配。
    • re.S：使 . 能匹配包括换行符在内的所有字符。
    • re.U：根据 Unicode 字符集解析字符。
    • re.X：为提高可读性，忽略空格和 # 后的注释。
  1. re.search re.search 函数会扫描整个字符串并返回第一个成功的匹配。
  • search 与 match 的区别：
    • match 要求从字符串的起始位置进行匹配。
import re

raw_str = 'abc12def'
result1 = re.search('\d+', raw_str)
result2 = re.match('\d+', raw_str)
print(result1.group())  # 12
print(result2.group())  # None

  1. re.sub 用于替换字符串中的匹配项。函数签名为 re.sub(pattern, repl, string, count, flags) 。

  • 参数说明：

    • pattern：匹配的正则表达式。
    • repl：替换的字符串，也可以是一个函数。
    • string：要匹配的字符串。
    • count：模式匹配后替换的最大次数，默认替换所有。
    • flags：标志位，用于控制正则表达式的匹配方式。

  • 简单替换示例：

import re

phone = "2024-959-459"

# 删除非数字'-'
num = re.sub(r'\D', phone)
print(num) # 2024959459

# 将匹配的数字乘以 2
def double(matched):

    return str(value * 2)

raw_str = 'A12B3C'

  1. 分组 在正则表达式中，group() 函数用于提取分组截获的字符串。

import re

print(result2.group())  # 0

# 四、正则表达式模式

  • ^：匹配字符串的开头。
  • $：匹配字符串的末尾。
  • .：匹配任意字符，但不包括换行符。当 re.DOTALL 标记被指定时，可以匹配包括换行符的任意字符。
  • [...]：表示一组字符，单独列出：[amk] 匹配 'a'、'm' 或 'k' 。
  • [^...]：匹配不在 [] 中的字符，例如：[^abc] 匹配除了 a、b、c 之外的字符。
  • re*：匹配 0 个或多个表达式。
  • re+：匹配 1 个或多个表达式。
  • re?：匹配 0 个或 1 个由前面的正则表达式定义的片段，采用非贪婪方式。
  • re{n}：精确匹配 n 个前面的表达式。例如，o{2} 不能匹配 "Bob" 中的 'o'，但能匹配 "food" 中的两个 'o' 。
  • re{n,}：匹配 n 个及以上前面的表达式。例如，o{2,} 不能匹配 "Bob" 中的 'o'，但能匹配 "food" 中的两个及以上 'o' 。
  • re{n,m}：匹配 n 到 m 次由前面的正则表达式定义的片段，采用贪婪方式。
  • a|b：匹配 a 或 b 。
  • (re)：对正则表达式进行分组并记住匹配的文本。
  • (?imx)：正则表达式包含三种可选标志：i、m 或 x 。只影响括号中的区域。
  • (?-imx)：正则表达式关闭 i、m 或 x 可选标志。只影响括号中的区域。
  • (?:re)：类似于 (...)，但不表示一个组。
  • (?imx:re)：在括号中使用 i、m 或 x 可选标志。
  • (?-imx:re)：在括号中不使用 i、m 或 x 可选标志。
  • (?#...)：注释。
  • (?=re)：前向肯定界定符。如果所含正则表达式在当前位置成功匹配则成功，否则失败。但一旦所含表达式已经尝试，匹配引擎不会回溯；模式的剩余部分仍会在界定符的右边尝试。
  • (?!re)：前向否定界定符。与肯定界定符相反；当所含表达式不能在字符串当前位置匹配时成功。
  • (?>re)：匹配的独立模式，省去回溯。
  • \w：匹配字母数字及下划线。
  • \W：匹配非字母数字及下划线。
  • \s：匹配任意空白字符，等价于 [\t\n\r\f\v] 。
  • \S：匹配任意非空白字符。
  • \d：匹配任意数字，等价于 [0-9] 。
  • \D：匹配任意非数字。
  • \A：匹配字符串开始。
  • \Z：匹配字符串结束，如果存在换行符，只匹配到换行前的结束字符串。
  • \z：匹配字符串结束。
  • \G：匹配最后匹配完成的位置。
  • \b：匹配一个单词边界，即单词和空格之间的位置。例如，'e' 可以匹配 "never" 中的 'er'，但不能匹配 "verb" 中的 'er' 。
  • \B：匹配非单词边界。'er\B' 能匹配 "verb" 中的 'er'，但不能匹配 "never" 中的 'er' 。
  • \n、\t 等：匹配一个换行符、制表符等。
  • \1...\9：匹配第 n 个分组的内容。
  • \10：匹配第 n 个分组的内容，如果它已匹配。否则指的是八进制字符码的表达式。

# 五、常用示例

  1. 匹配 "Python" 或 "python" ：[Pp]ython

  2. 匹配括号内的任意字母：[aeiou]

  3. 匹配任意字母或数字：[a-zA-Z0-9]

  4. 匹配除了数字外的字符：[^0-9] 或 \D

  5. 匹配邮箱：[\w\.-]+@\.\w+

  6. findall 多个匹配模式，返回元组列表

import re

result = re.findall(r'(\w+)=(\d+)', 'width=20,height=10')
print(result)  # [('width', '20'), ('height', '10')]

一个简单的实际案例

  1. 提取网页中的所有链接 假设我们有一段网页的 HTML 代码如下：
<p>这是一个示例页面，包含以下链接：<a href="https://www.example1.com">示例 1</a> 和 <a href="https://www.example2.com/page">示例 2</a></p>
1

我们可以使用正则表达式 href="(https?://[^\s]+)" 来提取出所有的链接。

  1. 验证密码强度 要求密码必须包含至少一个大写字母、一个小写字母、一个数字和一个特殊字符，长度在 8 到 16 位之间。可以使用正则表达式 ^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[@$!%*?&])[A-Za-z\d@$!%*?&]{8,16}$ 进行验证。

  2. 去除文本中的多余空格 如果有一个文本字符串 “ 这 是 一 个 有 很 多 空 格 的 文 本 ” ，我们可以使用正则表达式 \s+ 来匹配多个空格，然后用 re.sub 函数将其替换为一个空格，从而达到去除多余空格的目的。

  3. 提取文件名中的扩展名 例如文件名 “file.txt” ，使用正则表达式 \.\w+$ 可以提取出 .txt 这个扩展名。
