# django模型

在 Django 中，模型（Model）是用于定义数据库结构和与数据库进行交互的核心组件。 模型是通过继承 django.db.models.Model 类来创建的。每个模型代表数据库中的一个表，模型的属性对应表中的列。在 Django 中，模型（Model）是用于定义数据库结构和与数据库进行交互的核心组件。 模型是通过继承 django.db.models.Model 类来创建的。每个模型代表数据库中的一个表，模型的属性对应表中的列。

# Field常见参数

- max_length：字段最大长度，用于字符串等，字符串类型CharField必须设置该值
- null：如果True，Django将在数据库中存储NULL空值。默认是False
- blank：如果True，该字段被允许为空白("")。默认是False。请注意，这不同于null。null纯粹是与数据库相关的，而blank与验证相关。如果一个字段有blank=True，表单验证将允许输入一个空值。如果一个字段有blank=False，该字段将是必需的。
- choices：示例：YEAR_IN_SCHOOL_CHOICES = (('FR', 'Freshman'),('SO', 'Sophomore'),('JR', 'Junior'),('SR', 'Senior'),('GR', 'Graduate')) ,中文示例：SEX_CHOICES=((1, '男'),(2, '女'))，元组中的第一个元素是将存储在数据库中的值，第二个元素是将在页面中显示的值，最常见用于下拉选择框select
- default：字段的默认值
- help_text：用于显示额外的“帮助”文本
- primary_key：如果True，这个字段是模型的主键，默认是False
- unique：如果True，该字段在整个表格中必须是唯一的
- verbose_name：详细字段名，不指定则是属性名的小写，并且用 空格 替换 '_'

# 模型之间的关系

在 Django 中，模型之间主要有三种关系：一对多（OneToOneField、ForeignKey）、多对多（ManyToManyField）和一对一（OneToOneField）。

  1. 一对多关系（ForeignKey）：

    一个模型的实例可以与另一个模型的多个实例相关联。例如，一篇文章可以有多个评论，但一个评论只属于一篇文章。

    from django.db import models

class Article(models.Model):
    title = models.CharField(max_length=200)

class Comment(models.Model):
    article = models.ForeignKey(Article, on_delete=models.CASCADE)
    content = models.TextField()

    在上述示例中，Comment 模型通过 ForeignKey 与 Article 模型建立了一对多的关系。on_delete=models.CASCADE 表示当关联的 Article 被删除时，相关的 Comment 也会被自动删除。

  2. 多对多关系（ManyToManyField）：

    两个模型的实例之间可以存在多个对多个的关联。例如，一个学生可以选修多门课程，一门课程也可以有多个学生选修。

    class Student(models.Model):
    name = models.CharField(max_length=100)
    courses = models.ManyToManyField('Course')

class Course(models.Model):
    name = models.CharField(max_length=100)

# 其他相关知识点

  1. 模型的元数据（Meta）： 您可以在模型内部定义一个 Meta 类来提供关于模型的额外信息和配置选项。例如，设置排序规则、数据库表名、唯一性约束等。

    class MyModel(models.Model):
    # 模型字段

    class Meta:
        verbose_name = '自定义模型名称'
        verbose_name_plural = '自定义模型的复数名称'
        ordering = ['field_name']  # 按指定字段排序
        unique_together = [('field1', 'field2')]  # 联合唯一性约束

  2. 模型方法： 您可以为模型定义自定义方法，以执行与模型实例相关的操作。这些方法可以用于数据处理、计算属性等。

    class Book(models.Model):
    title = models.CharField(max_length=200)
    price = models.DecimalField(max_digits=5, decimal_places=2)

    def discount_price(self, discount_percentage):
        return self.price * (1 - discount_percentage / 100)

  3. 数据迁移： 当您对模型进行更改（添加、删除字段，修改字段属性等）时，需要使用 Django 的迁移功能来将这些更改应用到数据库中。迁移命令会自动生成数据库操作的脚本，并执行以保持数据库结构与模型的同步。

  4. 模型管理器（Manager）： 每个模型都有一个默认的管理器 objects，您可以自定义管理器来添加特定的查询方法或过滤条件。

    class BookManager(models.Manager):
    def expensive_books(self):
        return self.filter(price__gt=50)

class Book(models.Model):
    # 模型字段
    objects = BookManager()

  5. 模型继承： Django 支持三种模型继承方式：抽象基类继承、多表继承和代理模型继承。

    • 抽象基类继承：父类模型不会创建单独的数据库表，其字段会被添加到子类的表中。

    • 多表继承：父类和子类都会创建各自的数据库表，并且通过外键建立关联。

    • 代理模型继承：不改变数据库结构，只是为原始模型创建一个新的管理器。

  6. 模型验证： 您可以在模型中定义 clean 方法来执行自定义的验证逻辑。如果验证失败，可以抛出 ValidationError 异常。

  7. 模型与表单的集成： Django 的表单可以基于模型自动生成，方便数据的收集和验证。
