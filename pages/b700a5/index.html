# python算法

"""
常见的排序算法:
插入排序, 【*冒泡排序, *快速排序】
归并排序, 堆排序, 计数排序, 桶排序，希尔排序、选择排序

**复杂度：
1、时间复杂度；
2、空间复杂度

插入排序：O(n^2)
冒泡排序：O(n^2)
快速排序：O(nlogn)
堆排序：O(nlogn)
计数排序：O(n+k)
桶排序：O(n+k)
希尔排序：O(n^1.3)
选择排序：O(n^2)
冒泡排序：O(n^2)
快速排序：O(nlogn)
归并排序：O(nlogn)

常见的时间复杂度：
O(1): 常量阶，执行时间不随着某个变量的增长而增长，输入规模越小，花费的时间就越少
O(logN): 对数阶，执行时间随着某个变量的增长而减少，输入规模越小，花费的时间就越少
O(n): 线性阶，执行时间随着某个变量的增长而增加，输入规模越大，花费的时间就越多
O(nlogN): 线性对数阶，执行时间随着某个变量的增长而增加，输入规模越大，花费的时间就越多
O(n^2): 平方阶，执行时间随着某个变量的增长而增长，输入规模越大，花费的时间就越多
O(2^n): 指数阶，执行时间随着某个变量的增长而爆炸性增长，输入规模越大，花费的时间就越多
O(n!): 阶乘阶，执行时间随着某个变量的增长而增长极快，输入规模越大，花费的时间就越多


常见的时间复杂度：
O(1) < O(logN) < O(n) < O(nlogN) <  O(n^2) < O(2^n) < O(n!)
随着问题的规模n，不断增大，时间复杂度高的算法所花费的时间不断增大，意味算法的效率越低

时间复杂度分析统计的不是算法运行时间，而是算法运行时间随着数据量变大时的增长趋势
空间复杂度 space complexity」用于衡量算法占用内存空间随着数据量变大时的增长趋势
"""

# 冒泡排序的实现 本质是遍历数组，将最大的元素冒泡到最后
def bubble_sort(arr):
    n = len(arr)
    # 遍历所有数组元素
    for i in range(n):
        # 最后i个元素已经排好序，不需要再比较
        for j in range(0, n-i-1):
            # 交换如果元素是逆序的
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]
    return arr
"""
它的执行顺序是这样的：
首先，右侧的元组 arr[j+1], arr[j] 被评估。这意味着 arr[j+1] 的值被取出，然后 arr[j] 的值也被取出。这两个值组成了一个临时的元组。
接下来，这个临时元组被解包（unpacked），即它的元素被分别分配给左侧的变量。具体来说，arr[j+1] 的原始值被赋给 arr[j]，而 arr[j] 的原始值被赋给 arr[j+1]。
由于这两个赋值操作是同时进行的，因此不需要引入一个额外的临时变量来交换这两个值。
所以，这个过程实际上是在原地（in-place）交换了数组 arr 中索引为 j 和 j+1 的两个元素的值。
在冒泡排序的上下文中，这种交换操作用于将较大的元素“冒泡”到数组的右侧，从而实现排序。
要注意的是，在Python中，这种多重赋值的方式是原子性的（atomic），即在这个操作执行期间，
不会有其他线程或进程能够打断它。这保证了交换操作的正确性和一致性。
"""

"""
各类常见排序算法的时间复杂度和空间复杂度如下：
===========================================【1】========================================================
1、冒泡排序（Bubble Sort）：
--时间复杂度：平均情况下为O(n²)，最好情况下为O(n)（待排序元素为顺序排列时），最坏情况下也为O(n²)（待排序元素为逆序排列时）。
--空间复杂度：O(1)，因为冒泡排序是原地排序，不需要额外的存储空间。

2、选择排序（Selection Sort）：
--时间复杂度：平均和最坏情况下均为O(n²)。
--空间复杂度：O(1)，同样是原地排序。

3、插入排序（Insertion Sort）：
--时间复杂度：平均情况下为O(n²)，最好情况下为O(n)（待排序元素几乎有序时），最坏情况下也为O(n²)。
--空间复杂度：O(1)，也是原地排序。
============================================【2】=======================================================

4、快速排序（Quick Sort）：
--时间复杂度：平均情况下为O(n log n)，最好情况下也为O(n log n)，最坏情况下为O(n²)。
--空间复杂度：O(log n)（递归调用栈空间），但在最坏情况下可能会达到O(n)（当输入数组已经有序时）。

5、归并排序（Merge Sort）：
--时间复杂度：总是为O(n log n)，无论是最好、平均还是最坏情况。
--空间复杂度：O(n)，因为归并排序需要一个额外的数组来辅助排序。

6、堆排序（Heap Sort）：
--时间复杂度：O(n log n)。
--空间复杂度：O(1)，但需要考虑堆结构所需的额外空间，这通常可以忽略不计。
==============================================【3】==============================================

7、希尔排序（Shell Sort）：
--时间复杂度：依赖于增量序列的选择，但通常介于O(n)和O(n²)之间。
--空间复杂度：O(1)，因为它是原地排序。
==============================================【4】=============================================

8、计数排序（Counting Sort）、桶排序（Bucket Sort）和基数排序（Radix Sort）：
这些排序算法的时间复杂度依赖于输入数据的分布和范围。在特定情况下，它们可以达到线性时间复杂度O(n)。
空间复杂度通常高于O(1)，因为它们需要额外的存储空间来存储计数、桶或基数
"""


"""
案例：快速排序
时间复杂度：平均情况下为O(nlogn)，最好情况下也为O(nlogn)，最坏情况下为O(n²)。
空间复杂度：O(log n)（递归调用栈空间），但在最坏情况下可能会达到O(n)（当输入数组已经有序时）。
快速排序原理：
1、选择基准元素：选择一个基准元素（pivot），通常可以选择待排序序列的第一个或最后一个元素作为基准。
2、分区（Partition）：将待排序序列分为两个子序列，使得左子序列中的所有元素都小于等于基准元素，右子序列中的所有元素都大于基准元素。
3、递归排序左子序列和右子序列：递归地对左子序列和右子序列进行快速排序，直到子序列长度为1，即无法再分。
"""
def quick_sort(arr):
    if len(arr) <= 1:
        return arr
    else:
        pivot = arr[0]  # 选择第一个元素作为基准
        less = [x for x in arr[1:] if x <= pivot]  # 小于等于基准的子序列
        greater = [x for x in arr[1:] if x > pivot]  # 大于基准的子序列
        return quick_sort(less) + [pivot] + quick_sort(greater)

quick_list = [3,99,6,55, 8, 10, 1, 2, 1]
print(quick_sort(quick_list))


"""
案例： 插入排序
实现原理：
1、将待排序序列分为已排序部分和未排序部分。初始时，已排序部分只包含第一个元素，而未排序部分包含剩余的元素。
2、从未排序部分中取出第一个元素，将其插入到已排序部分的适当位置，使得插入后的已排序部分仍然保持有序。
3、重复上述步骤，依次将未排序部分的元素取出并插入到已排序部分的适当位置，直到未排序部分为空。
"""
def insert_sort(arr):
    for i in range(1, len(arr)):
        key = arr[i] # 当前待插入的元素
        j = i - 1    # 已排序部分的最后一个元素的索引

        #  # 将比 key 大的元素向右移动
        while j >= 0 and arr[j] > key:
            arr[j+1] = arr[j]
            j -= 1
        arr[j+1] = key
    return arr

# 测试示例
arr = [7, 2, 1, 6, 8, 5, 3, 4]
insert_sort(arr)
print(arr)


def merge_sort(list_data):
    if len(list_data) <= 1:
        return list_data
    count = int (len(list_data)/2)
    left  = merge_sort(list_data[:count])
    right = merge_sort(list_data[count:])
    return merge(left,right)

def merge(left,right):
    l,r = 0,0
    result=[]
    while l < len(left) and r<len(right):
        if left[l] < right[r]:
            result.append(left[l])
            l = l+1 
        else:
            result.append(right[r])
            r += 1
    result += list(left[l:])
    result += list(right[r:])
    return result
if __name__ == "__main__":
    nums = [10,1,35,61,89,36,55] 
    print(merge_sort(nums))

'''
归并排序（Merge Sort）是一种基于分治思想的排序算法。它的核心思想是将两个或两个以上的有序表合并成一个新的有序表。具体步骤如下：
分解：将当前区间一分为二，然后再分解这两个子区间，如此递归进行，直到区间长度为1，此时认为区间内已经达到有序。
解决：将两个有序的子区间进行合并，得到完全有序的区间。
合并：将各个有序的子区间逐步合并，最终得到完全有序的序列。
'''

def selector_sort(nums):
    for j in range(0,len(nums)-1): 
        min_index = j
        for i in range(min_index+1,len(nums)):
            if nums[i] < nums[min_index]:
                min_index = i
        nums[j],nums[min_index] = nums[min_index],nums[j]

def select_sort_max(nums):
    for j in range(0,len(nums)):
        max_idx = j
        for i in range(max_idx+1,len(nums)):
            if nums[i] > nums[max_idx]:
                max_idx = i
        nums[j],nums[max_idx] = nums[max_idx],nums[j]
    return nums

if __name__ == "__main__":
    nums = [10,1,35,61,89,36,55]
    select_sort_max(nums)
    print(nums)
'''
选择排序的原理是每一次从待排序的数据元素中选出最小（或最大）的一个元素，存放在序列的起始位置，直到全部待排序的数据元素排完。
排序过程中，最小（或最大）元素的选择需要进行比较，比较次数与序列的初始状态有关。因此，选择排序的最大和最小排序次数是不确定的，
需要根据具体序列的初始状态来确定。一般情况下，选择排序的时间复杂度为O(n^2)，其中n为序列的长度
'''

"""
拓展：
【选择排序的原理】：
1、遍历待排序序列，找到最小（或最大）的元素。
2、将找到的最小（或最大）元素与待排序序列的第一个元素交换位置。
3、在剩余的待排序序列中重复上述步骤，每次找到最小（或最大）的元素，并将其与待排序序列的第一个未排序元素交换位置。
4、重复执行上述步骤，直到待排序序列为空。

【冒泡排序的原理】：

1、从待排序序列的第一个元素开始，依次比较相邻的两个元素。
2、如果前一个元素大于后一个元素，则交换它们的位置。
3、继续向后遍历，重复执行上述比较和交换步骤，将最大的元素逐渐冒泡到序列的末尾。
4、重复执行上述步骤，直到待排序序列已完全排序。

【**区别】：
1、排序方式：选择排序通过找到最小（或最大）元素并进行交换，将最小（或最大）元素放到正确的位置。
冒泡排序通过相邻元素的比较和交换，将较大的元素逐渐冒泡到序列的末尾。

2、循环次数：选择排序需要进行 n-1 次循环，其中 n 是待排序序列的长度。
           冒泡排序需要进行 n-1 轮比较和交换，每轮都将最大的元素放到末尾。
           
3、性能表现：选择排序的平均和最坏时间复杂度都是 O(n^2)，无论输入数据的情况如何。
冒泡排序的平均和最坏时间复杂度也都是 O(n^2)。但是在实际应用中，冒泡排序通常比选择排序慢，因为冒泡排序每次都只交换相邻的两个元素，
而选择排序可以通过一次交换来减少交换次数。

5、稳定性：选择排序是不稳定的排序算法，相等元素的相对顺序可能发生改变。冒泡排序是稳定的排序算法，相等元素的相对顺序不会改变。

综上所述，选择排序和冒泡排序在实现和性能方面有一些区别，但都属于简单的排序算法，适用于小规模的数据集。
对于大规模数据，更高效的排序算法如快速排序和归并排序通常更为合适。
"""
